Charte CSS

Nous codons pour des humains, pour nos collaborateurs présents et/ou futurs donc un code lisible et compréhensible est primordial. Cette charte propose des règles pour écrire des CSS cohérents.

Nommage

  • Codez le plus possible en anglais (noms des classes, commentaires)
  • Ecrivez en minuscule, séparez les mots par un tiret, pas de CamelCase
  • N'utilisez qu'un type de tiret : tiret “-” et non tiret bas “_”
  • Utilisez toujours le même type de guillemets

Format


.selector-1,
.s-2,
aside .s-3 {
    position: relative;
    display: inline-block;
    padding: 6px 50px;
    margin: 0 auto 20px;
    font-size: 18px;
    line-height: 1.3em;
    background-color: #aaa;
    border: 1px solid #efefef;
    -webkit-transition: all .2s;
    -moz-transition:    all .2s;
    -ms-transition:     all .2s;
    -o-transition:      all .2s;
    transition:         all .2s;
}
  • N'utilisez pas d’ID en CSS, uniquement des classes
  • tiret “-” et pas d’underscore “_”
  • Un seul sélecteur par ligne
  • Un espace avant l'accolade ouvrante d'une règle
  • Un espace après les deux points d'une déclaration
  • Une déclaration par ligne dans un bloc de déclarations
  • Ne précisez pas les unités pour les valeurs nulles
  • Utilisez le code hexadecimal à 3 caractères quand cela est possible
  • Avec les préfixes vendeurs, l'alignement vertical se fait après les deux points
  • Ne précisez pas le 0 dans une valeur comme 0.2
  • Ajoutez toujours un point-virgule à la fin d'une déclaration
  • Fermez l'accolade d'une règle au même niveau que le premier caractère de la règle
  • Sautez une ligne entre chaque règle

Exceptions et légers écarts


.selector-1 { width: 10%; }
.selecteur-2 { width: 20%; }
.selecteur-3 { width: 30%; }

De gros blocs de déclarations uniques peuvent utiliser un format légèrement différent, à savoir d'être regroupées sur une seule ligne. Dans ce cas, il faut insérer un espace après l'accolade ouvrante et avant l'accolade fermante.


.selecteur {
    box-shadow:
        1px 1px 1px #000,
        2px 2px 1px 1px #ccc inset;
    background-image:
        linear-gradient(#fff, #ccc),
        linear-gradient(#f3c, #4ec);
}

Les longues valeurs de propriétés séparées par des virgules - comme des ensembles de dégradés et d'ombre - peuvent être agencées sur plusieurs lignes de manière à améliorer la lisibilité et produire des fichiers diff plus utiles. Il existe plusieurs formats que l'on peut utiliser.
