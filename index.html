Charte CSS

Nous codons pour des humains, pour nos collaborateurs présents et/ou futurs donc un code lisible et compréhensible est primordial. Cette charte propose des règles pour écrire des CSS cohérentes et uniformisées pour améliorer leur maintenabilité.

Structure

Il est préférable de répartir les CSS en catégories dans des fichiers séparés. Les fichiers sont à charger dans l'ordre pour respecter la cascade des styles.

Nommage

Écrivez toujours en minuscule et séparez les mots par un tiret, que ce soit pour les classes CSS ou les images, pas de CamelCase. Le nom d'une classe doit signifier la fonction de l'élément qu'elle style et non son apparence. Codez le plus possible en anglais (noms des classes, commentaires).

Format


.selector-1,
li + li,
.s-2,
nav > .s-3 {
    position: relative;
    display: inline-block;
    padding: 6px 50px;
    margin: 0 auto 20px;
    font-size: 18px;
    line-height: 1.3em;
    background-color: #aaa;
    border: 1px solid #efefef;
    -webkit-transition: all .2s;
    -moz-transition:    all .2s;
    -ms-transition:     all .2s;
    -o-transition:      all .2s;
    transition:         all .2s;
}
 
.selector a {
    color: #bada55;
}
  • N'utilisez pas d’ID en CSS, uniquement des classes.
  • tiret “-” et pas tiret bas (underscore) “_”.
  • Un seul sélecteur par ligne.
  • Un espace avant l'accolade ouvrante d'une règle.
  • Un espace après les deux points d'une déclaration.
  • Une déclaration par ligne dans un bloc de déclarations.
  • Ne précisez pas les unités pour les valeurs nulles.
  • Utilisez le code hexadécimal à 3 caractères dès que possible.
  • Avec les préfixes vendeurs, l'alignement vertical se fait après les deux points.
  • Ne précisez pas le 0 dans une valeur comme 0.2.
  • Sautez une ligne entre chaque règle.
  • Ajoutez toujours un point-virgule à la fin d'une déclaration.
  • Fermez l'accolade au même niveau que les sélecteurs.

Exceptions


.selector-1 { width: 10%; }
.selector-2 { width: 20%; }
.selector-3 { width: 30%; }

Des blocs de déclarations courtes peuvent utiliser un format différent sur une seule ligne. Dans ce cas, il faut insérer un espace après l'accolade ouvrante et avant l'accolade fermante.


.selector {
    box-shadow:
        1px 1px 1px #000,
        2px 2px 1px 1px #ccc inset;
    background-image:
        linear-gradient(#fff, #ccc),
        linear-gradient(#f3c, #4ec);
}

Les longues valeurs de propriétés séparées par des virgules peuvent être agencées sur plusieurs lignes de manière à améliorer la lisibilité et produire des fichiers diff plus utiles.
